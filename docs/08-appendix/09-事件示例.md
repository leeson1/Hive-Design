# 09 事件示例

## Purpose

- 给出跨对象、跨状态的完整运行示例。
- 帮助实现方理解事件、对象、状态迁移和恢复如何协同。

## 示例 1：用户一句话新建项目

### 输入

- 用户输入：`做一个带认证和权限模型的服务骨架`

### 事件序列

1. `UserInputReceived`
2. `RuntimeDirectiveCreated`
3. `ResearchRequested`
4. `PlanCompiled`
5. `TaskCreated`
6. `TaskQualified`
7. `TaskDispatched`
8. `AgentRunStarted`

### 对象变化

- 生成 `Directive`
- 生成 `Research Sprint`
- 生成 `Evidence Pack`
- 编译 `Brief`
- 编译 `Charter`
- 编译 `Execution Plan`
- 编译 `Task Graph`
- 创建首批 `Task` 与 `AgentRun`

### 结果

- 项目从单句目标进入可调度状态。

## 示例 2：运行中用户纠偏导致 Task Supersession

### 输入

- 用户新增输入：`先别做移动端，优先把认证链路做完整`

### 事件序列

1. `UserInputReceived`
2. `RuntimeDirectiveCreated`
3. `PlanRevised`
4. `IssueOpened`（若存在冲突）
5. `TaskCreated`
6. `TaskQualified`

### 对象变化

- `Directive` 进入 impact analysis
- `Execution Plan` 从 `plan_rev_11` 更新到 `plan_rev_12`
- `task_mobile_ui_01` 进入 `superseded`
- 活跃 `AgentRun` 被标记为 `finish_current_step`
- 新任务 `task_auth_backend_07` 创建并 ready

### 结果

- 旧工作线被安全收束。
- 新工作线接管执行。

## 示例 3：Worker Timeout 后恢复并重派

### 输入

- `run_codex_003` 心跳停止

### 事件序列

1. `AgentRunHeartbeatMissed`
2. `AgentRunTimedOut`
3. `IssueOpened`
4. `RecoveryStarted`
5. `CheckpointWritten`
6. `TaskDispatched`
7. `AgentRunStarted`
8. `RecoveryCompleted`

### 对象变化

- `run_codex_003` 状态变为 `timed_out`
- `task_auth_backend_07` 不变为 completed，而是进入 recovery / reassign 路径
- 如果存在 partial handoff，则产物被回收
- 新 `AgentRun` 创建为 `run_codex_004`

### 结果

- 任务不中断为“已完成”假象。
- 系统通过恢复和重派继续推进。
