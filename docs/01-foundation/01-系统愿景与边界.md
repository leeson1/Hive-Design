# 01 系统愿景与边界

## 系统愿景

Hive 是一个**面向长期自主工作的、事件驱动的 agent 调度与项目状态控制系统**，由可替换的 AI 执行器驱动。

它本身不实现 agent，而是编排诸如 Claude Code、Codex 这类外部执行器，把用户输入转化为可审计、可恢复、可重规划的执行过程。

## 系统目标

- 用户可以只输入一句目标，系统自动展开研究、设计、规划与执行。
- 支持多个 Worker 并行运行，并允许随时替换异常或超时实例。
- 支持用户在运行中随时追加指令，并触发计划更新与任务调整。
- 支持总指挥周期性清理上下文后，依靠持久化状态对象恢复工作。

## 非目标（明确排除）

- 不是具体的 agent 产品本身
- 不是多 Agent 群聊系统
- 不是依赖长上下文记忆的协作系统
- 不是“谁聪明谁拍板”的自治系统

## 边界声明

- Hive 关注“项目状态转移”而非“单次对话质量”。
- Hive 负责调度、验收、恢复与重规划，不负责实现具体执行器。
- Worker 是外部可替换执行单元，不是项目事实来源。
- 用户输入必须先转化为显式的 Directive 或等价状态对象，才能影响运行态。
- 持久化对象是事实来源，文本对话不是事实来源。
