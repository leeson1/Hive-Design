# 01 Project Charter 规范（稳定层）

## Purpose

- 定义稳定层规则对象。
- 约束什么内容允许进入 Charter，什么内容不得进入。

## Rules

### Charter 必备内容

- 项目目标
- 不可违反约束
- 架构原则
- 术语定义
- 关键风险边界

### Charter Discipline

- Charter 低频变更。
- 运行排序、阶段次序、任务拆分不得写入 Charter。
- 若运行中纠偏触及目标边界或硬约束，必须经过 Decision 再更新 Charter。

## 变更门槛

- 必须记录 Decision
- 必须评估影响范围
- 必须同步 Checkpoint

## Acceptance Criteria

- Charter 中不存在临时执行排序。
- 任一 Charter 变更都能回到 Decision 与 Checkpoint。
